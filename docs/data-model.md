# Data Model — Simple Web Calculator

> **Architecture note:** All arithmetic is handled **client-side** in the browser.
> The database is intentionally minimal — it only supports optional history
> logging and key-value app configuration.

## Table Descriptions

### `calculation_history`

| Column       | Type     | Constraints     | Description                        |
|--------------|----------|-----------------|------------------------------------|
| `id`         | TEXT     | PK              | Unique record ID (cuid)            |
| `expression` | TEXT     | NOT NULL        | Human-readable expression string   |
| `result`     | TEXT     | NOT NULL        | Computed result as string          |
| `created_at` | DATETIME | DEFAULT now()   | Timestamp of the calculation       |

**Index:** `idx_calculation_history_created_at` on `created_at` for fast
recent-history queries.

## Design Decisions

| Decision | Rationale |
|---|---|
| **SQLite** as default DB | Zero-config, perfect for a simple app with no concurrency needs |
| **No User model** | App requires no authentication per the spec |
| **Results stored as TEXT** | Avoids floating-point precision loss at the DB layer |
| **History is optional** | Core feature is client-side; history is a nice-to-have log |
| **No relations** | Tables are independent; keeps schema dead simple |

## Data Flow

Browser (Next.js)
    │
    │  User clicks buttons → expression built in React state
    │  User presses "=" → result computed via eval/parser (client)
    │
    ├──[optional]──► POST /api/history  ──► calculation_history table
    │
    └──[on load]───► GET  /api/config   ──► app_config table
